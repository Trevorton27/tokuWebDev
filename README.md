# Signal Works LMS

An AI-powered Learning Management System with adaptive intake assessment, GitHub activity tracking, Google Calendar integration, and instructor-managed curriculum delivery.

## Features

- **Course & lesson management** ‚Äî Instructors create courses and lessons; students enroll and track progress
- **Intake assessment** ‚Äî Multi-step questionnaire scores students and generates a personalized AI roadmap
- **AI roadmap** ‚Äî Generated by Claude (Anthropic) and stored as a Google Doc assigned to each student
- **GitHub integration** ‚Äî Per-student webhook + PAT to track commits and repo activity
- **Google Calendar sync** ‚Äî Students and instructors connect their calendars to the LMS
- **Messaging** ‚Äî In-app conversations between students and instructors
- **Knowledge RAG** ‚Äî Semantic search across lesson content using Voyage AI embeddings
- **Admin dashboard** ‚Äî View all students, assessment results, roadmaps, and engagement metrics

## Getting Started

### Prerequisites

- Node.js 18+
- PostgreSQL database (Neon recommended)
- Clerk account
- Anthropic API key
- Voyage AI API key

### Installation

1. **Install dependencies**
   ```bash
   npm install
   ```

2. **Set up environment variables** ‚Äî see `env-vars.md` for the full reference:
   ```bash
   cp .env.example .env.local
   ```

   Minimum required variables:
   ```
   DATABASE_URL=
   NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=
   CLERK_SECRET_KEY=
   NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
   NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
   NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/student
   NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/student
   ANTHROPIC_API_KEY=
   ENCRYPTION_SECRET=       # 32-byte hex string
   ```

3. **Set up the database**
   ```bash
   npx prisma generate
   npx prisma migrate dev
   ```

4. **Run the development server**
   ```bash
   npm run dev
   ```

   Open [http://localhost:3000](http://localhost:3000).

## Environment Variables

See [`env-vars.md`](env-vars.md) for a full reference of every variable, its purpose, and which integrations require it.

**Required for core functionality:**

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | Neon/PostgreSQL connection string |
| `CLERK_SECRET_KEY` + `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Authentication |
| `ANTHROPIC_API_KEY` | AI roadmap generation |
| `ENCRYPTION_SECRET` | Encrypts GitHub tokens and webhook secrets at rest |

**Required per integration:**

| Integration | Variables |
|---|---|
| GitHub tracking | `GITHUB_PERSONAL_ACCESS_TOKEN`, `GITHUB_WEBHOOK_SECRET` |
| Google Calendar | `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_REDIRECT_URI` |
| Google Docs (roadmaps) | `GOOGLE_SERVICE_ACCOUNT_EMAIL`, `GOOGLE_PRIVATE_KEY` |
| Knowledge search | `EMBEDDINGS_API_KEY`, `EMBEDDINGS_API_URL`, `EMBEDDINGS_MODEL` |
| Cross-app (Assessment) | `NEXT_PUBLIC_ASSESSMENT_URL`, `ASSESSMENT_API_URL`, `INTERNAL_API_SECRET` |

## API Routes

### LMS
- `GET /api/lms/courses` ‚Äî List available courses
- `GET /api/lms/courses/[id]` ‚Äî Course details + lessons
- `POST /api/lms/enrollments` ‚Äî Enroll in a course
- `DELETE /api/lms/enrollments/unenroll` ‚Äî Unenroll

### GitHub
- `GET/PUT /api/github/profile` ‚Äî Get/set GitHub username
- `GET/POST/DELETE /api/github/pat` ‚Äî Manage Personal Access Token
- `GET/POST/DELETE /api/github/token` ‚Äî Manage webhook secret
- `POST /api/webhooks/github/[userId]` ‚Äî Per-user webhook endpoint

### Roadmap
- `GET /api/roadmap/document` ‚Äî Fetch student's assigned Google Docs roadmap
- `GET /api/roadmap/route` ‚Äî Fetch LMS-generated roadmap

### Google Calendar
- `GET /api/google-calendar/auth` ‚Äî Start OAuth flow
- `GET /api/google-calendar/callback` ‚Äî OAuth callback
- `GET /api/google-calendar/events` ‚Äî List synced events

### Admin
- `GET /api/admin/students/[studentId]/overview` ‚Äî Full student dashboard data
- `PUT /api/admin/students/[studentId]/roadmap-document` ‚Äî Assign Google Doc roadmap

### Knowledge
- `POST /api/knowledge/search` ‚Äî Semantic search
- `POST /api/knowledge/ingest` ‚Äî Index document (admin only)

## GitHub Webhook Setup (per student)

Students configure their own webhook so the LMS can track activity:

1. Go to **Student ‚Üí GitHub ‚Üí Webhook tab**
2. Copy the unique webhook URL shown in the UI
3. In GitHub: **Settings ‚Üí Webhooks ‚Üí Add webhook**
   - Payload URL: paste the unique URL
   - Content type: `application/json`
   - Secret: enter any secret string, then paste it back into the LMS webhook form
4. Save on GitHub, then save the secret in the LMS

The student can also add a Personal Access Token (PAT) to increase the GitHub API rate limit from 60 to 5,000 requests/hour.
